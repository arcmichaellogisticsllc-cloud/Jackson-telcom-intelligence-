# Operator Pilot Guide

## 30-Day Goal

Prove Jackson operators can use the platform with real data without drowning in modules, raw signals, or noisy recommendations.

## Daily Mike Workflow

1. Open Jackson Telcom Command Center.
2. Review Top 5 Actions.
3. Review Southeast work, capacity gaps, and relationship priorities.
4. Log calls, notes, follow-ups, and outcomes.
5. Submit Pilot Feedback when a page is confusing, noisy, missing data, or useful.
6. Resolve or assign Data Quality issues tied to Southeast records.

## Daily Ron Workflow

1. Open Jackson Telcom Command Center.
2. Review Top 5 Actions.
3. Review Great Lakes work, capacity gaps, and relationship priorities.
4. Log relationship actions and workforce/competitive movement notes.
5. Submit Pilot Feedback on bad priorities or missing context.
6. Resolve or assign Great Lakes Data Quality issues.

## Weekly Review

1. Run the sequential acquisition cycle.
2. Review Data Quality Review.
3. Review Connector Runs.
4. Review recommendation tuning rules.
5. Review SyncERP contract validation items.
6. Review audit logs for denied access, failed logins, and unusual activity.

## Logging Conversations

Use record action buttons for notes, calls, email drafts, meetings, and follow-ups. Email and messages are drafts only. The platform does not send outreach.

## Reporting Bad Data

Use Data Quality Review for:

- Duplicate entities
- Missing contact info
- Bad imports
- Low confidence signals
- Disputed classifications
- Stale contacts
- Missing region or owner

Resolve issues only after the linked record has been corrected, dismissed, or assigned.

## Decision Visuals

Use Decision Visuals for executive decisions only:

- Where should we invest?
- Who should we recruit?
- What should we pursue?
- Which relationships should we strengthen?
- What should we avoid?

Do not use visuals as raw reports.

## Avoid Module Hopping

Start with Command Center, Production Readiness, Decision Visuals, and the record detail page linked from the action. Modules feed intelligence; workspaces organize action.
